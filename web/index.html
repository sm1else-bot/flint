Why Flint

Built for people who know what they want.

Every feature exists because someone needed it, not because a product manager needed a slide.

Network-level ad blocking
Blocks at the WebView2 layer before requests leave the machine. Uses the StevenBlack unified hosts list, refreshed every seven days. On by default.
Complete privacy
No analytics. No crash reports. No accounts. The only external connections are pages you navigate to and optional weather data.
Chrome extensions
Load any unpacked Chrome extension from a local folder. Manifest V2 and V3 both supported. Each extension gets an Open button for its popup.
Glass-themed window
Mica and Acrylic backdrop via DWM. Semi-transparent glass panels. The Pegboard canvas inherits the window's backdrop so tiles float on blurred desktop.
Keyboard-first shortcuts
A low-level keyboard hook intercepts shortcuts before Chromium sees them. They work inside pages, text fields, and captchas without exception.
Address bar autocomplete
Dark glass dropdown with up to six history matches. Favicon, title, and dimmed URL for each result. Arrow keys navigate, Enter confirms, Escape closes.
Privacy

Everything stays on your machine.

Flint makes no external connections except the ones you explicitly trigger. Your browsing history, bookmarks, and settings live in a single JSON file at %LocalAppData%\Flint\profile.json.

  • No analytics or crash reporting
  • No account sign-in or sync
  • No background update processes
  • No sidebar AI or suggestions
What does connect

Only when you use it.

These are the only outbound connections Flint makes, all opt-in or user-triggered.

  • Pages you navigate to
  • Favicon lookups via Google favicon service
  • Ad-block list refresh from GitHub (every 7 days)
  • Open-Meteo weather data (only if you add a weather tile)
